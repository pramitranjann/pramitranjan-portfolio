Albers

Key Insight_
“When the brief said ‘interaction of colors,’ I took the word interaction literally.”
Problem_
I took “interaction of color” literally and built an app around it.

Albers is a macOS color theory tool named after Josef Albers. It isolates perceptual phenomena like simultaneous contrast and optical mixing, giving users real-time controls to manipulate each one directly.

My Role_
Sole designer and developer, from concept to shipped macOS app.
Solution_
Nine modes, one premise: color has no fixed identity.

Modes: Shade, Tint, Tone, Harmony, Relativity, Bezold, Optical Mix, Transparency, and Visual Vibration. CAM maps the selected color into hue, chroma, and lightness values so perception can be read analytically as well as visually.

Process_

The brief said “interaction of colors.” As a UX major, that sounded like a UI problem.

Figma handled visual direction; iteration happened directly in the app using vibe coding. AI handled implementation while I focused on what the interaction should feel like, making the gap between design and build much smaller.

Research_
Color had to be something people could manipulate, not just study.

Most responses to the assignment went editorial. I kept returning to Albers’s argument that color has no fixed identity. That felt like a software problem as much as an art one.

Challenge_
A proof of concept became a product with a real identity.

The first submission was just a working prototype. Spring break became the space to push it into a more complete tool, adding modes, tightening the interaction model, and developing the amber HUD aesthetic.

Solution_

Nine modes. One premise: color has no fixed identity.

A nine-mode macOS application where each view isolates one perceptual color phenomenon with live controls. The amber HUD frames the app as an instrument of observation, not a creative canvas.
